A tiny interactive demo that shows why `0.1 + 0.2 !== 0.3` in programming, and what that tells us about how computers handle numbers.

## What this is

This project demonstrates a common surprise in programming: computers sometimes produce results that look mathematically wrong. The goal is to show _why this happens_ in the simplest way possible, without heavy theory or jargon.

## How it works

- Enter two numbers
- Click **Add**
- The result is shown exactly as the computer calculates it

Try:

```txt

```
